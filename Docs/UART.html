UART Driver Documentation

The Universal Asynchronous Receiver-Transmitter (UART) module provides fully interrupt-driven, bidirectional serial communication for the Test Bench. It ensures zero data loss during high-speed communication with Electronic Control Units (ECUs).

Hardware Configuration

The STM32F401 Blackpill uses USART1 (PA9/PA10) for the Zephyr Shell by default. Therefore, the Test Bench leverages USART2 and USART6 for automated testing channels.

Channel 1 (USART2) TX: PA2 | RX: PA3
Channel 2 (USART6) TX: PA11 | RX: PA12
Default Baudrate 115200 bps
Architecture Note: This driver uses 256-byte Zephyr ring_buf software buffers and hardware Interrupt Service Routines (ISRs). When data arrives on the RX pin, the ISR instantly pushes it to the Ring Buffer without blocking the main CPU execution.

Zephyr Shell API Commands

1. Send String

Transmits a string of characters out the TX pin.

blackpill:~$ uart send <channel> <message>
Example: 
blackpill:~$ uart send 1 START_ENGINE

2. Read Buffer

Drains the software ring buffer, printing all characters received since the last read. If no data has arrived, it returns silently.

blackpill:~$ uart read <channel>
Example: 
blackpill:~$ uart read 2
ENGINE_RUNNING_OK

3. Clear Buffer

Resets the RX ring buffer. This is highly recommended before starting a new automated Robot Framework test to ensure garbage data is flushed.

blackpill:~$ uart clear <channel>

4. Dynamically Set Baudrate

Reconfigures the hardware UART peripheral to a new speed on the fly, without needing to reboot the Test Bench.

blackpill:~$ uart set_baud <channel> <baudrate>
Example: 
blackpill:~$ uart set_baud 1 9600
