PWM IN Driver Documentation

The PWM IN Driver for the Hardware-in-the-Loop (HIL) Test Bench enables the Zephyr RTOS shell to capture and analyze incoming PWM signals. It utilizes hardware timer interrupts to precisely measure the period and pulse width of an external signal.

Hardware Configuration

The Test Bench leverages Hardware Timer 3 on the STM32F401 Blackpill to capture signals. The timer is configured with a Prescaler of 83 to slow down the 84 MHz system clock to 1 MHz (1 tick = 1 microsecond), preventing 16-bit timer overflow at low frequencies.

Channel 1 Pin: PB4
Channel 2 Pin: PB5
Timer Core TIM3 (16-bit)
Timer Clock 1 MHz

Zephyr Shell API Commands

The driver exposes three powerful commands to the Zephyr shell under the pwm_in subsystem.

1. Read (Frequency & Duty Cycle)

Captures both the frequency and duty cycle simultaneously using a single hardware capture request.

blackpill:~$ pwm_in read <channel_number>
Example: 
blackpill:~$ pwm_in read 1
Freq: 1000 Hz, Duty: 25.00

2. Get Frequency Only

Optimizes the capture flags to only measure the period of the signal, returning the frequency.

blackpill:~$ pwm_in get_frequency <channel_number>
Example: 
blackpill:~$ pwm_in get_frequency 2
Freq: 10000 Hz

3. Get Duty Cycle Only

Measures both parameters internally but only outputs the calculated duty cycle percentage.

blackpill:~$ pwm_in get_dutyCycle <channel_number>
Example: 
blackpill:~$ pwm_in get_dutyCycle 1
Duty: 75.00

Performance Statistics & Driver Precision

Extensive statistical profiling (~10,000 samples) was conducted to map the precision of the STM32F401 TIM3 capture driver across various frequencies. Due to the 16-bit timer architecture and the selected prescaler, precision scales inversely with frequency.

Operating Range: The absolute hardware limit is 16 Hz to 500 kHz. Frequencies below 16 Hz will cause the 16-bit timer to overflow.

Architectural Note

The Test Bench guarantees < 1% margin of error for all signals up to 10 kHz, making it highly optimal for standard Automotive and Industrial control signals. For high-frequency RF or fast switching signals (>10 kHz), transitioning to the 32-bit TIM5 core with a 0-prescaler configuration is recommended.
